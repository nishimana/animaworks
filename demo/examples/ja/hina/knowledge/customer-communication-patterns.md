# 顧客コミュニケーションパターン

> 作成日: 2026-03-02
> 作成者: Hina（カスタマーサクセス / アシスタント）
> 目的: 顧客対応の優先度判断と通知テンプレートを標準化
> 更新: 2026-03-02 15:30 — テナント分離バグ対応テンプレートを追加

## 優先度判断マトリックス

| 優先度 | 事象 | 初動時間 | 対応手順 |
|--------|------|---------|---------|
| P0 緊急 | データ漏洩・セキュリティ | 即時 | Soraに技術調査依頼 → 修正確認後に顧客通知 |
| P1 重大 | 機能停止・データ不整合 | 1時間以内 | 影響範囲特定 → 暫定通知 → 修正後に最終通知 |
| P2 通常 | パフォーマンス劣化・UI不具合 | 4時間以内 | 再現確認 → Soraにエスカレーション |
| P3 軽微 | 質問・要望 | 翌営業日 | FAQ確認 → 回答 or ナレッジ化 |

## コミュニケーションチャネル使い分け

| チャネル | 用途 | 例 |
|---------|------|-----|
| メール | 正式通知、障害報告、リリース通知 | セキュリティ修正のお知らせ |
| チャット | クイック対応、質問回答 | 「エクスポートできない」→ 手順案内 |
| Board | チーム内共有 | 対応完了報告 |

## 通知テンプレート

### テンプレート1: リリース通知

```
件名: PixelForge {version} リリースのお知らせ

{顧客名}様

PixelForge {version} をリリースしましたのでお知らせいたします。

【主な更新内容】
{更新内容リスト}

【セキュリティ修正】
{セキュリティ修正があれば記載}

ご不明点がございましたら、お気軽にお問い合わせください。

NovaCraft サポートチーム
```

### テンプレート2: 障害発生通知（初報）

```
件名: 【重要】PixelForge サービスに関するお知らせ

{顧客名}様

現在、PixelForgeにおいて以下の事象が発生しております。

【事象】
{事象の概要 — 技術用語を避け、顧客影響の観点で記述}

【影響範囲】
{影響を受ける機能・ユーザー}

【対応状況】
現在、技術チームが原因調査および修正作業を行っております。
修正完了次第、改めてご連絡いたします。

ご迷惑をおかけし、誠に申し訳ございません。
```

### テンプレート3: 障害復旧通知

```
件名: 【復旧完了】PixelForge サービス復旧のお知らせ

{顧客名}様

先ほどお知らせいたしました事象について、修正が完了いたしました。

【原因】
{根本原因 — Soraの報告を顧客向けに翻訳}

【修正内容】
{修正の概要}

【データへの影響】
{データの変更・削除・漏洩の有無}

ご迷惑をおかけし、申し訳ございませんでした。
今後このような事象が発生しないよう、再発防止策を実施いたします。
```

### テンプレート4: 機能要望への返信

```
件名: Re: {機能要望タイトル}

{顧客名}様

ご要望をいただきありがとうございます。

{要望内容}について、開発チームに共有いたしました。
v{次バージョン}のロードマップ検討時に優先度を評価させていただきます。

進展がございましたら改めてご連絡いたします。
```

```

直ちに修正対応を完了いたしましたのでお知らせいたします。

【事象】
{事象の概要}

【影響】
{影響範囲 — 「データの変更・削除は発生しておりません」等}

【対応】
{修正・デプロイ完了の旨}

【再発防止策】
{技術的な再発防止策の概要}

ご不安な点がございましたら、いつでもお問い合わせください。
```

## Sora→Hina 引き渡しパターン

障害時にSoraから受け取る情報と、顧客向けに変換するフロー:

```
Sora の報告:
  「design-cache.ts のキャッシュキーにテナントIDが欠落。
   cache.get(designId) → cache.get(tenantId:designId) に修正」

↓ 顧客向け変換:

顧客への通知:
  「キャッシュシステムの不具合により、一部のお客様において
   他アカウントのデザインが一時的に閲覧可能な状態となっておりました。
   修正を完了し、デプロイ済みです。
   データの変更・削除は発生しておりません。」
```

## v0.3障害対応の教訓

- 影響顧客の特定に10分かかった → 顧客DBとの連携があれば即時化できる
- テンプレート5（セキュリティ緊急通知）が不足していた → 追加した
- 全5社から受領確認を取るまで追跡した → 追跡プロセスも標準化する
