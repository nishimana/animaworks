## Overview

v0.5 represents a **leap in production readiness** for AnimaWorks.
Through fundamental improvements to the memory system, multi-layered security hardening,
a complete Workspace UI overhaul, and major streaming performance gains,
this release establishes the foundation for Animas to operate reliably
as autonomous individuals in real-world environments.

### Highlights

- **Memory system overhaul** — Distilled Knowledge removed in favor of full RAG. Episode memory channel added
- **Multi-layered security** — Data provenance tracking (4 phases complete), encrypted credential vault
- **Workspace reimagined** — Canvas node graph, live activity cards, KPI dashboard
- **Streaming performance** — RAF batching + incremental Markdown rendering for 3–5x perceived speedup
- **Operational stability** — Config hot-reload, progress-aware hang detection, LLM API auto-retry

## 1. Memory System Evolution

### Distilled Knowledge Removed → Full RAG Migration

Previously, a fixed budget of knowledge was "distilled" and injected into the system prompt.
This approach suffered from inefficient context allocation and scalability limits.

v0.5 **expands Channel C (related knowledge) budget** so RAG vector search now covers
knowledge, procedures, and common knowledge comprehensively.
DK is fully removed, consolidating everything into the Priming layer's dynamic retrieval.

### Channel F: Episode Memory

A 6th Priming channel for **episode memory search** (400 tokens) has been added.
`search_memory` now also supports `scope="episodes"`,
enabling direct lookup of "when did I do what?"

### Spreading Activation Repair

A bug in the NetworkX-based graph spreading activation caused some nodes to be
unreachable during search. This has been fixed, and episode node support has been added,
improving memory graph coverage.

### Consolidation Quality Improvements

- Auto-repair of broken YAML frontmatter in knowledge files
- Automatic extraction and memorization of `[REFLECTION]...[/REFLECTION]` blocks from heartbeats
- Enhanced activity log noise filtering (deduplication, short-text skip)

### Search Accuracy

- `search_memory` OR-split queries — compound questions decomposed into parallel keyword searches
- Improved keyword extraction accuracy for Priming Channel C
- Knowledge frontmatter validation on write and startup

## 2. Multi-Layered Security

### Data Provenance Tracking — 4 Phases Complete

A **trust chain** has been established for external data entering the memory system.

| Phase | Description |
|-------|-------------|
| Phase 1 | `resolve_trust()` foundation, origin constants, wrapper extensions |
| Phase 2 | Automatic origin tagging at external data entry points (Slack, Chatwork, web search, Gmail) |
| Phase 3 | `origin_chain` propagation in Anima-to-Anima messaging (tracks original trust through relays) |
| Phase 4 | Origin metadata on RAG chunks, trust-separated Priming output |

When `origin_chain` contains `external_platform`,
the entire message is treated as **untrusted even if the relaying Anima is trusted**.

### Encrypted Credential Vault

- PyNaCl SealedBox encryption (`vault.json`)
- Animas can operate via `vault_get` / `vault_store` / `vault_list` tools
- Notification channel credentials also support vault / shared credential resolution

### Board Channel ACL

Access control introduced for shared channels.
Read/write permissions can be restricted per Anima per channel.

### Other Security Fixes

- Path traversal prevention in `common_knowledge`, `create_anima`, and `sender_profile`
- Network tools added to command blocklist, newline injection prevention
- Inbox `from_person` validation, channel POST authentication

## 5. Execution Engine

### Linear Prompt Budget

The 4-tier system (T1–T4) has been replaced with **linear budget allocation
proportional to context window size**. This makes more efficient use of model capacity.

### Config Hot-Reload

Changes to `config.json` and `status.json` are **applied without process restart**.
IPC reload commands instantly update connections, schedules, and model settings.

### Progress-Aware Hang Detection

Counter-based kill logic has been replaced.
LLM response progress (token output, tool execution) is monitored,
and timeouts trigger **only when the process is genuinely stalled**.

### LLM API Auto-Retry

Automatic retry for 429 (rate limit), 5xx (server error), and network errors
across all execution modes. Transient failures no longer stall Animas.

### Background Model

Heartbeat / Inbox / Cron can now run on a **separate lightweight model**.
Example: Main Opus 4.6 + Background Sonnet 4.6 for ~80% cost reduction.

```bash
animaworks anima set-background-model alice claude-sonnet-4-6
```

### Other

- `animaworks anima rename` command
- `--force` flag for stop/restart
- Auto-save `recovery_note.md` on heartbeat crash
- Unified tool visibility across S/A/B modes
- Qwen 3.5 model support (Mode A + 64K context)
- Codex SDK (Mode C) stability improvements

## 10. Breaking Changes

### Distilled Knowledge Removed

The DK section in `builder.py` is removed.
Channel C RAG search covers all knowledge — no migration required.

### Tier-Based Prompt Scaling Removed

T1–T4 tiered system prompt has been replaced with linear budget allocation.
No configuration changes needed (auto-applied).

### status.json as SSoT

`supervisor` / `speciality` fields now use `status.json` as the sole source of truth.
The `config.json` `animas` section is limited to organization layout only.
