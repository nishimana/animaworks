# AnimaWorks v0.5 リリースノート

> v0.4.5 (2026-02-28) → v0.5.1 (2026-03-06) の全変更をカバー

## 概要

v0.5 は AnimaWorks にとって**運用品質の飛躍**を意味するリリースです。
記憶システムの根本的な改善、セキュリティの多層強化、Workspace UI の全面刷新、
そしてストリーミング性能の大幅改善を通じて、
「自律的な人格」としてのAnimaが実戦環境で安定稼働するための基盤を整えました。

### ハイライト

- **記憶システム刷新** — Distilled Knowledge を廃止し RAG 全面移行。エピソード記憶チャネル追加
- **セキュリティ多層化** — データ出自追跡（Provenance）4フェーズ完了、資格情報の暗号化保管
- **Workspace 全面刷新** — キャンバスノードグラフ、ライブアクティビティ、KPIダッシュボード
- **ストリーミング高速化** — RAF バッチング＋インクリメンタル Markdown で体感速度 3〜5 倍改善
- **運用安定性** — 設定ホットリロード、進捗認識型ハング検知、LLM API 自動リトライ

## 1. 記憶システムの進化

### Distilled Knowledge 廃止 → RAG 全面移行

従来はシステムプロンプトに固定量の知識を「蒸留」して注入していましたが（Distilled Knowledge）、
コンテキスト予算の非効率さとスケーラビリティの限界が顕著でした。

v0.5 では **Channel C（関連知識）のバジェットを拡張**し、RAG ベクトル検索が
知識・手順・共有知識すべてをカバーするように変更。
DK を完全に廃止し、Priming レイヤーの動的検索に一本化しました。

### Channel F: エピソード記憶の追加

Priming の 6 番目のチャネルとして**エピソード記憶検索**（400 トークン）を新設。
`search_memory` にも `scope="episodes"` を追加し、
「いつ何をしたか」を直接検索できるようになりました。

### 拡散活性化の修復

NetworkX ベースのグラフ拡散活性化にバグがあり、一部のノードが検索にヒットしない問題を修正。
エピソードノードのサポートも追加し、記憶グラフの網羅性が向上しました。

### 記憶統合の品質向上

- knowledge ファイルのフロントメタデータ（YAML）破損を自動修復
- Heartbeat の `[REFLECTION]...[/REFLECTION]` ブロックを自動抽出して記憶化
- アクティビティログのノイズフィルタリング強化（重複排除・短文スキップ）

### 検索精度の改善

- `search_memory` の OR 分割クエリ — 複合質問を個別キーワードに分解して並列検索
- Priming Channel C のキーワード抽出精度向上
- 知識フロントメタデータの書き込み時・起動時バリデーション

## 2. セキュリティの多層強化

### データ出自追跡（Provenance）— 4 フェーズ完遂

外部から取得したデータが記憶に混入する際の**信頼性チェーン**を確立しました。

| フェーズ | 内容 |
|---------|------|
| Phase 1 | `resolve_trust()` 基盤関数、origin 定数、ラッパー拡張 |
| Phase 2 | 外部データ入口（Slack, Chatwork, Web検索, Gmail）に origin タグ自動付与 |
| Phase 3 | Anima 間メッセージで `origin_chain` を伝播（中継されても元の trust を追跡） |
| Phase 4 | RAG チャンクに origin メタデータを記録、Priming 出力を trust レベル別にセパレート |

`origin_chain` に `external_platform` が含まれる場合、
中継 Anima が trusted でも**全体を untrusted として扱う**ルールを徹底。

### 資格情報の暗号化保管（Credential Vault）

- PyNaCl SealedBox による暗号化（`vault.json`）
- Anima から `vault_get` / `vault_store` / `vault_list` ツールで操作可能
- 通知チャネルの資格情報も vault / shared credential に対応

### Board チャネル ACL

共有チャネルにアクセス制御を導入。チャネルごとに読み書き可能な Anima を制限。

### その他のセキュリティ修正

- `common_knowledge`、`create_anima`、`sender_profile` のパス・トラバーサル防止
- コマンドブロックリストにネットワークツール追加、改行インジェクション対策
- Inbox の `from_person` バリデーション、チャネル POST 認証

## 3. Workspace UI の全面刷新

### キャンバスノードグラフ

従来のリスト型組織図を廃止し、**ノードグラフ形式のキャンバスレイアウト**に移行。
Anima 間の関係をビジュアルに把握でき、ドラッグ操作で視点を変更可能。

### ライブアクティビティカード

各 Anima のカードに**リアルタイムのツール実行状況**を表示。
ハートビート中・チャット応答中・タスク実行中の状態がアニメーションで即座にわかります。

### KPI ダッシュボード

Workspace トップに KPI パネルを追加。ポーリングによるライブ更新で
稼働中 Anima 数、タスク完了率、エラー率などを一目で確認。

### メッセージライン＆アバターバリアント

組織図上に Anima 間のメッセージフローを可視化するラインを追加。
アバターも状態に応じたバリアント（通常・作業中・休止中）を表示。

## 4. チャット体験の向上

### ストリーミング高速化（体感 3〜5 倍）

| 施策 | 効果 |
|------|------|
| requestAnimationFrame バッチング | DOM 更新を描画フレームに同期、ジャンク削減 |
| インクリメンタル Markdown レンダリング | 30ms / 10文字間隔で差分のみ再レンダリング |
| Zone-based 部分 DOM 更新 | チャットバブル内のみ書き換え、全体再描画を回避 |
| ASGI ミドルウェア最適化 | SSE フラッシュ間隔の短縮 |
| ログ出力削減 | ストリーミング中の不要なログ I/O を抑制 |

### マルチペインチャット

VS Code 風にチャット画面を**左右に分割**して複数 Anima と同時会話。
ペインごとに独立したスレッド・ストリーミング状態を管理。

### コピー＆ダウンロードボタン

アシスタントのチャットバブルに**コピー・ダウンロード**のアクションボタンを追加。
コードブロックやレスポンス全体をワンクリックで取得。

### ツールコール折りたたみ

チャット内のツール実行履歴を**展開可能なグループ**にまとめ、
会話の流れを阻害しない UI に改善。

### モバイル UX

- アイコンのみ表示時のアニマ名ツールチップ
- 折りたたみ可能なアニマタブ
- スクロール可能なスレッドタブ
- ボタンレイアウトの一貫性向上

## 5. 実行エンジンの強化

### リニアプロンプトバジェット

従来の 4 段階ティア（T1〜T4）を廃止し、**コンテキストウィンドウサイズに比例した
リニアバジェット配分**に移行。モデルの能力をより効率的に活用します。

### 設定ホットリロード

`config.json` や `status.json` の変更を**プロセス再起動なし**で反映。
IPC 経由の reload コマンドで接続・スケジュール・モデル設定が即座に更新されます。

### 進捗認識型ハング検知

従来のカウンタベースの kill 判定を廃止。
LLM の応答進捗（トークン出力・ツール実行）を監視し、
**実際に止まっている場合のみ**タイムアウトする賢い検知方式に移行。

### LLM API 自動リトライ

429（レート制限）、5xx（サーバーエラー）、ネットワークエラーに対する
自動リトライを全実行モードに導入。一時的な障害で Anima が止まらなくなりました。

### バックグラウンドモデル

Heartbeat / Inbox / Cron を**メインモデルとは別の軽量モデル**で実行可能に。
例: メイン Opus 4.6 + バックグラウンド Sonnet 4.6 でコスト 80% 削減。

```bash
animaworks anima set-background-model alice claude-sonnet-4-6
```

### その他

- `animaworks anima rename` コマンド追加
- `--force` フラグで stop/restart を強制実行
- Heartbeat クラッシュ時に `recovery_note.md` を自動保存
- ツール可視性を S/A/B 全モードで統一
- Qwen 3.5 モデルサポート（Mode A + 64K コンテキスト）
- Codex SDK (Mode C) の安定性向上

## 6. 組織とコミュニケーション

### Slack 統合の大幅強化

- **スレッドコンテキスト注入** — Slack スレッド内のメッセージ受信時に直近のスレッド文脈を自動注入
- **Per-Anima アイコン・名前** — 各 Anima が Slack に固有のアイコン URL と表示名で投稿
- **Per-Anima Bot トークン** — Anima ごとに別々の Bot トークンを使用可能
- **app_mention イベント対応** — `ts` ベースの重複排除＋スレッド返信コンテキスト
- **intent 自動判定** — Slack メンション / DM で即時 inbox 処理トリガー

### 外部メッセージ受信の改善

- 受信メッセージに**返信指示メタデータ**を自動注入（返信先チャネル・スレッドの自動解決）
- `call_human` の返信を**発信元 Anima に自動ルーティング**
- 外部メッセージの重複排除

### タスク管理の強化

- Heartbeat / Inbox で**タスク作成を MUST 指示**に昇格
- DAG スケジューラによる**並列タスク実行**（`plan_tasks`）
- `plan_tasks` を MCP 経由で公開、Heartbeat プロンプトで推奨
- タスク失敗時の安全なファイルライフサイクルと障害通知
- `human` ソースタスクのボーナス優先度

### スーパーバイザーツール

- `audit_subordinate` — 部下の活動サマリー・エラー頻度・ツール使用統計
- 全スーパーバイザーツールを Mode S MCP で公開
- CLI `audit` サブコマンド追加

## 10. 破壊的変更

### Distilled Knowledge の廃止

`builder.py` の DK セクションは廃止済み。
Channel C の RAG 検索が全知識をカバーするため、移行作業は不要です。

### ティア型プロンプトスケーリングの廃止

T1〜T4 の段階的システムプロンプトを廃止し、リニアバジェット配分に移行。
設定変更は不要（自動適用）。

### status.json の SSoT 化

`supervisor` / `speciality` フィールドは `status.json` が唯一の正（SSoT）に。
`config.json` の `animas` セクションは組織レイアウトのみに限定されます。

## アップグレード手順

```bash
pip install --upgrade animaworks

# 設定の互換性チェック
animaworks config check

# サーバー再起動
animaworks restart --force
```

特別な移行作業は不要です。
DK 廃止とリニアバジェットは自動適用されます。
