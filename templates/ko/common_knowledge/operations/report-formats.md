# 보고서 형식 모음

각 역할이 사용하는 표준 보고 템플릿과 체크리스트입니다.

## 매니저: 상황 보고

```markdown
## 상황 보고

### 완료 사항
- [완료한 태스크와 성과]

### 진행 중
- [태스크명]: [진행률/상태] — [다음 단계]

### 과제 및 리스크
- [과제 내용과 영향도] — [대응 방침]

### 판단이 필요한 사항
- [판단을 요청할 내용과 선택지]
```

## 리서처: 조사 보고서

```markdown
# 조사 보고서: [주제]

## 개요
[1~3문장으로 조사 결과 요약]

## 조사 목적
[무엇을 밝히려 했는가]

## 조사 방법
[어떻게 조사했는가 — 검색 키워드, 참조처]

## 발견 사항
### 주요 발견
- [항목별 나열]

### 상세
[각 발견 사항의 상세 설명]

## 출처 및 신뢰도
| 출처 | 유형 | 신뢰도 |
|------|------|--------|
| [URL/이름] | [공식/1차/2차] | [높음/중간/낮음] |

## 결론 및 권고
[조사 결과에 기반한 판단과 다음 액션]
```

## 운영: 정기 모니터링 보고서

```markdown
## 정기 모니터링 보고서

### 확인 시각
[YYYY-MM-DD HH:MM]

### 시스템 상태
- [대상]: [정상/주의/이상] — [보충 설명]

### 리소스 현황
- 디스크: [사용률]%
- 메모리: [사용률]%

### 최근 이벤트
- [이벤트 요약]

### 후속 조치
- [대응이 필요한 경우의 내용]
```

## 운영: 인시던트 기록

```markdown
## 인시던트 기록

- 발생 시각: [YYYY-MM-DD HH:MM]
- 감지 방법: [heartbeat/cron/수동]
- 심각도: [P1~P4]
- 영향 범위: [구체적인 영향]
- 원인: [확인된 원인 / 조사 중]
- 대응 내용: [실시한 대응]
- 재발 방지: [필요한 대책]
```

## 라이터: 셀프 리뷰 체크리스트

### 내용
- [ ] 목적이 서두에 명확히 기술되어 있는가
- [ ] 논리적 비약이 없는가
- [ ] 독자의 배경 지식에 맞는 설명 수준인가
- [ ] 필요한 정보가 빠져 있지 않은가
- [ ] 불필요한 정보가 포함되어 있지 않은가

### 표현
- [ ] 한 문장이 너무 길지 않은가 (목표: 약 60자 이내)
- [ ] 같은 내용의 반복이 없는가
- [ ] 피동태가 과도하지 않은가 (능동태 우선)
- [ ] 모호한 표현을 구체화할 수 있지 않은가

### 형식
- [ ] 제목 계층이 정리되어 있는가
- [ ] 항목 나열의 세분화 수준이 일관적인가
- [ ] 코드 블록, 링크, 표가 올바르게 렌더링되는가
- [ ] 오탈자가 없는가
