# 유스케이스: 커뮤니케이션 자동화

외부 채팅 도구나 이메일의 모니터링 및 대응을 자동화하는 유스케이스입니다.

---

## 이런 과제를 해결할 수 있습니다

- 채팅이나 이메일 응답이 지연되기 쉬움
- 여러 channel을 동시에 감시하기 어려움
- 중요한 메시지를 놓쳐버림
- 야간/휴일의 연락에 대응할 수 없음
- 외부 파트너에게 정기 연락을 잊어버림

---

## 패턴 1: 채팅 도구 정기 모니터링

### 하는 일

Anima가 일정 간격(예: 30분마다)으로 채팅 도구의 새 메시지를 확인하고 내용에 따라 대응합니다.

### 동작 흐름

1. 정기 순찰로 지정 channel의 새 메시지를 가져옴
2. 메시지 내용을 분석 (긴급도, 대응 필요 여부 판단)
3. 대응 패턴에 따라 분기:
   - **스스로 대응 가능한 내용** → 직접 답신
   - **사람의 판단이 필요** → 사람에게 알리고 판단을 요청
   - **다른 담당자 대상** → 사내 적절한 Anima에 전달
   - **대응 불필요** → 로그에 기록하고 다음으로

### 활용 예시

- 거래처의 납기 확인에 "확인 후 회신하겠습니다"라고 즉시 답변
- 고객의 긴급 연락을 감지하여 사람에게 알림
- 자주 묻는 질문에 템플릿 답변을 자동 전송

### 시작하기 위해 필요한 것

- 채팅 도구의 API 연동 설정
- 감시 대상 channel 및 그룹 지정
- 대응 규칙 (무엇을 자동 답신하고, 무엇을 에스컬레이션할지)

---

## 패턴 2: 이메일 대응 자동화

### 하는 일

수신 이메일을 정기적으로 확인하고 분류, 답신, 전달을 자동화합니다.

### 동작 흐름

1. 정기적으로 메일함을 확인
2. 제목, 발신자, 본문에서 내용을 분류:
   - 청구서/영수증 → 경리 담당에 전달
   - 문의 → 고객 지원 담당에 전달
   - 영업 메일/스팸 → 무시 (로그만)
   - 중요한 연락 → 사람에게 알림
3. 필요에 따라 수령 확인 이메일을 자동 전송

### 활용 예시

- "견적 요청" 이메일 수신 → 사람에게 "견적 요청이 왔습니다"라고 알림
- 외부 서비스의 알림 이메일 → 내용을 요약하여 모니터링 담당에 전달
- 정형 문의 → 템플릿 답신을 임시 저장 (사람이 확인 후 전송)

### 주의사항

- 자동 전송 이메일의 범위는 신중하게 설정하세요
- 처음에는 "임시 저장까지"로 하고 전송은 사람이 확인하는 운용이 안전합니다
- 중요한 거래처의 이메일은 반드시 사람에게 알립니다

---

## 패턴 3: 에스컬레이션 자동화

### 하는 일

여러 channel(채팅, 이메일, SNS 등)을 종합적으로 감시하고 중요도에 따라 적절한 상대에게 에스컬레이션합니다.

### 동작 흐름

1. 각 channel을 정기 감시
2. 메시지의 중요도를 판정:
   - **긴급** (서비스 장애, 불만 등) → 즉시 사람에게 알림
   - **중요** (계약/금전 관련 내용) → 다음 순찰 시 사람에게 보고
   - **일반** (일상적 문의) → Anima가 대응 시도
   - **낮음** (정보 공유만) → 로그에 기록
3. 에스컬레이션 시 요약과 권장 조치를 첨부

### 활용 예시

- 심야에 서버 장애 알림 도착 → 사람의 스마트폰에 긴급 알림
- 거래처로부터 계약 내용 변경 연락 → 다음 날 아침 사람 출근 시 요약 제시
- 직원의 휴가 신청 → 캘린더에 임시 등록하고 사람에게 승인 요청

---

## 패턴 4: 정기 연락 자동화

### 하는 일

정해진 타이밍에 정해진 상대에게 정형 연락을 자동 전송합니다.

### 동작 흐름

1. 스케줄 설정 (매일/매주/매월 등)
2. 지정 시각에 메시지를 작성
3. 지정 channel/상대에게 전송
4. 전송 결과를 로그에 기록

### 활용 예시

- 매일 아침 9시에 "오늘의 일정 요약"을 채팅에 게시
- 매주 금요일에 "이번 주 활동 리포트"를 관계자에게 전송
- 월말에 "청구서 송부 리마인더"를 경리 담당에게 알림
- 프로젝트 진행 상황을 주간으로 관계자에게 자동 보고

---

## 패턴 5: 다국어 커뮤니케이션 지원

### 하는 일

외국어 메시지를 수신했을 때 번역과 요약을 자동으로 수행합니다.

### 동작 흐름

1. 외국어 메시지를 감지
2. 내용을 번역 및 요약
3. 사람에게 "이러한 내용의 메시지가 왔습니다"라고 모국어로 보고
4. 필요에 따라 답신 임시 저장을 외국어로 작성

### 활용 예시

- 해외 거래처로부터의 영어 이메일 → 한국어로 요약하여 보고
- 해외 SNS에서의 자사 언급 → 번역하여 공유
- 다국어 문의 → 각 언어로 답신안을 작성

---

## 구성 힌트

### 최소 구성 (Anima 1개)

- 1개로 여러 channel을 감시
- 단순한 규칙 기반으로 대응을 분기
- 판단에 망설여지면 전부 사람에게 에스컬레이션

### 권장 구성 (Anima 2~3개)

- **감시 및 분배 담당**: 전체 channel을 감시하고 내용을 분류
- **대응 담당**: 분류된 메시지에 대해 답신 및 처리를 실행
- 역할을 나눔으로써 감시 누락과 대응 품질 양쪽을 담보

### 대규모 구성

- channel별로 전담 감시 Anima를 배치
- 총괄 Anima가 전체를 조정
- 사람은 총괄 Anima의 요약만 확인하면 됨
