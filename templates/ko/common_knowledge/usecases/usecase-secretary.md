# 유스케이스: 비서 및 사무 지원

스케줄 관리, 연락 조정, 리마인더, 정보 정리 등 일상의 사무 작업을 자동화하는 유스케이스입니다.

---

## 이런 과제를 해결할 수 있습니다

- 일정 관리가 따라잡지 못함
- 회의나 마감일을 잊어버림
- 관계자에게 연락 및 조정에 시간을 빼앗김
- 일보/주보 작성이 번거로움
- 여러 프로젝트의 진행 상황을 파악하기 어려움

---

## 패턴 1: 스케줄 관리 및 리마인더

### 하는 일

캘린더를 감시하고, 일정의 리마인드와 빈 시간 관리를 수행합니다.

### 동작 흐름

1. 매일 아침, 당일과 다음 날의 일정을 가져옴
2. 사람에게 "오늘의 일정 요약"을 전송
3. 각 일정 시작 전(예: 30분 전)에 리마인더를 전송
4. 일정의 중복이나 빈 시간 부족을 감지하여 경고

### 활용 예시

- 매일 아침 8시에 "오늘의 일정은 3건입니다. 10시: ○○ 회의, 14시: ○○ 면담, 16시: ○○ 마감"
- 회의 30분 전에 "곧 ○○ 회의입니다. 자료는 여기입니다"
- "내일 오전에 빈 시간이 없습니다. ○○ 일정을 옮기시겠습니까?"

### 응용

- 캘린더에 새 일정이 추가되면 관련 자료를 사전에 준비
- 정기 회의 전에 전번 의사록과 이번 안건을 정리하여 송부
- 이동 시간을 고려한 현실적인 스케줄 제안

## 패턴 2: 연락 조정 자동화

### 하는 일

여러 관계자와의 일정 조정이나 연락 사항의 전달을 대행합니다.

### 동작 흐름

1. 사람이 "○○ 씨와 다음 주 미팅을 잡아 줘"라고 지시
2. 사람의 캘린더에서 빈 시간을 추출
3. 후보 일시를 정리하여 관계자에게 연락
4. 답변을 집약하여 최적의 일시를 제안
5. 확정되면 캘린더에 등록

### 활용 예시

- "다음 주 중에 ○○ 팀과 미팅을 잡아 줘" → 후보 3개를 제시
- 외부 거래처에 "다음 정기 회의 일정을 확인하고 싶습니다"라고 이메일
- 참석자가 많은 회의의 일정 조정을 자동으로 수행하고, 결정되면 전원에게 알림

### 주의사항

- 외부 연락은 사람의 확인 후 전송하는 것이 안전합니다
- 중요한 상담의 일정 조정은 최종 확인을 사람에게 요청합니다

## 패턴 3: 일보 및 리포트 자동 작성

### 하는 일

하루의 활동 로그와 성과를 집약하여 정형 리포트를 자동 생성합니다.

### 동작 흐름

1. 지정 시각(예: 매일 17시)에 그날의 활동 데이터를 수집
2. 송수신 메시지, 완료 태스크, 발생 이벤트를 정리
3. 템플릿에 따라 리포트를 작성
4. 사람에게 리뷰용으로 전송 (또는 자동 게시)

### 활용 예시

- 매일 업무 일보를 자동 생성
- 주간 프로젝트 진행 요약을 작성
- 월말에 당월 활동 통계 리포트를 생성

### 템플릿 예시

```
== 오늘의 활동 요약 ==
■ 완료 태스크: 5건
  - 태스크 A (완료)
  - 태스크 B (완료)
  ...
■ 수신 메시지: 12건 (대응 완료 10건, 보류 2건)
■ 발생 이벤트: 특이사항 없음
■ 내일 일정: 3건
```

## 패턴 4: 정보 수집 및 브리핑

### 하는 일

아침 첫 시간에 필요한 정보를 모아 간결한 브리핑으로 전달합니다.

### 동작 흐름

1. 아침 지정 시각에 각종 정보를 수집:
   - 미읽 메시지 요약
   - 오늘의 일정
   - 진행 중 태스크의 상황
   - 뉴스나 시장 데이터 (설정이 있는 경우)
2. 우선순위순으로 정리
3. "좋은 아침입니다. 오늘의 브리핑입니다"로 전송

### 활용 예시

- "어젯밤 사이에 중요한 메시지가 2건 왔습니다"
- "오늘의 최우선 태스크는 ○○ 마감입니다"
- "어제 매출은 ○○원이었습니다. 전주 대비 +5%"

## 패턴 5: 태스크 관리 및 진행 추적

### 하는 일

태스크의 등록, 진행 추적, 리마인더를 일원 관리합니다.

### 동작 흐름

1. 사람의 지시나 메시지에서 태스크를 추출 및 등록
2. 기한에 기반하여 리마인더를 설정
3. 정기적으로 진행 상황을 확인 및 보고
4. 기한 초과 태스크를 경고

### 활용 예시

- "다음 주 금요일까지 견적서를 보내 줘" → 태스크 등록 + 금요일 아침에 리마인드
- "○○ 진행 상황을 주간으로 확인해 줘" → 매주 월요일에 상태 확인 + 보고
- 기한 3일 전, 1일 전, 당일에 각각 리마인더

---

## 패턴 6: 경비 및 청구서 관리 보조

### 하는 일

청구서나 경비 정산의 정리 및 리마인드를 지원합니다.

### 동작 흐름

1. 이메일이나 채팅으로 도착한 청구서를 감지
2. 금액, 결제 기한, 발송처를 추출하여 목록화
3. 결제 기한 전에 리마인더를 전송
4. 월말에 미처리 청구서 목록을 보고

### 활용 예시

- "이번 달 미결제 청구서는 3건, 합계 ○○원입니다"
- "○○사의 청구서 결제 기한이 3일 후입니다"
- 월간 경비 목록을 자동 작성

## 구성 힌트

### 최소 구성 (Anima 1개)

- 1개의 "비서 Anima"가 전체를 겸임
- 스케줄 관리 + 메시지 감시 + 리마인더가 기본 세트
- 이것만으로도 "항상 누군가가 지켜보고 있다는 안심감"을 얻을 수 있음

### 권장 구성

- **비서 Anima**: 스케줄, 연락, 리마인더
- **리포트 Anima**: 일보, 주보, 진행 리포트 자동 생성
- 2개의 분업으로 비서의 부하를 분산

### 효과적인 운용 요령

- 처음에는 리마인더만으로도 충분한 효과가 있습니다
- "사람이 잊기 쉬운 것"부터 자동화해 나갑니다
- 중요한 연락 대행은 처음에 "임시 저장 → 사람 확인 → 전송"의 흐름으로
- 익숙해지면 점차 자동화 범위를 넓힙니다
