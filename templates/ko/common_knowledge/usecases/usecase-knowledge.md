# 유스케이스: 지식 관리 및 문서 정비

절차서 작성 및 업데이트, FAQ 정비, 정보 구조화 등 조직의 지식을 관리하고 발전시키는 유스케이스입니다.

---

## 이런 과제를 해결할 수 있습니다

- 절차서가 오래된 채 업데이트되지 않음
- 같은 질문에 여러 번 답하고 있음
- 속인화된 지식이 공유되지 않음
- 문서가 산재하여 찾을 수 없음
- 신규 멤버에게 인수인계하는 데 시간이 걸림

---

## 패턴 1: 절차서 자동 작성 및 업데이트

### 하는 일

실제 작업 로그에서 절차서를 자동 생성하고 변경이 있으면 업데이트합니다.

### 동작 흐름

1. 작업 실행 로그를 감시
2. 반복 수행되는 작업 패턴을 감지
3. 절차서 초안을 자동 생성:
   - 전제 조건
   - 절차 (단계별)
   - 주의사항 및 자주 발생하는 에러
   - 완료 조건
4. 사람의 리뷰 후 정식 버전으로 저장

### 활용 예시

- 서버 배포 절차를 실행 로그에서 자동 생성
- 트러블슈팅 대응 이력을 FAQ화
- 새 도구의 설정 절차를 첫 작업 시 기록

### 포인트

- "같은 패턴이 3번 나오면 절차서화"가 적절한 단위
- 절차서에는 "왜 이렇게 하는지"의 배경 설명도 포함
- 실행 시 문제가 발생하면 피드백으로 절차서에 반영

---

## 패턴 2: FAQ 구축과 자동 답변

### 하는 일

자주 묻는 질문과 답변을 데이터베이스화하고, 같은 질문이 오면 자동 답변합니다.

### 동작 흐름

1. 문의 이력을 분석
2. 빈출 질문 패턴을 특정
3. 답변 템플릿을 작성
4. 새로운 문의가 오면:
   - FAQ에 해당하는 질문이 있으면 자동 답변
   - 없으면 사람에게 에스컬레이션
   - 사람이 답변하면 해당 Q&A를 FAQ에 추가

### 활용 예시

- "비밀번호 재설정 방법은?" → FAQ에서 자동 답변
- "○○ 기능의 사용법은?" → 관련 절차서 링크를 제시
- 새로운 질문이 옴 → 사람의 답변을 기록하여 FAQ에 추가

## 패턴 3: 지식의 구조화 및 분류

### 하는 일

산재하는 정보를 체계적으로 정리하여 검색하기 쉬운 형태로 정리합니다.

### 동작 흐름

1. 기존 문서, 메모, 채팅 로그를 수집
2. 내용을 카테고리별로 분류
3. 중복이나 모순을 감지
4. 구조화된 목차(인덱스)를 작성
5. 정기적으로 목차를 업데이트

### 활용 예시

- 프로젝트의 설계 문서를 테마별로 정리
- 채팅에서 공유된 지견을 카테고리별로 정리
- 여러 파일에 산재하는 정보를 통합하여 1개의 가이드로

---

## 패턴 4: 교훈 및 회고의 축적

### 하는 일

문제가 발생했을 때의 원인과 대책을 "교훈"으로 기록하여 같은 문제의 재발을 방지합니다.

### 동작 흐름

1. 문제/장애 발생
2. 대응 완료 후 다음을 기록:
   - 무엇이 발생했는지 (현상)
   - 왜 발생했는지 (근본 원인)
   - 어떻게 대처했는지 (대응 절차)
   - 향후 어떻게 방지할 것인지 (재발 방지책)
3. 유사 문제 발생 시 과거의 교훈을 자동 검색하여 제시

### 활용 예시

- "이전에 유사한 에러가 발생한 적이 있습니다. 전번 대처법: ○○"
- 장애 대응 후 "포스트모템 리포트"를 자동 생성
- 분기별로 "자주 발생하는 문제와 대책 일람"을 업데이트

## 패턴 5: 온보딩 자료 정비

### 하는 일

새로운 멤버(사람이든 Anima이든)가 조직에 합류할 때의 인수인계 자료를 정비하고 업데이트합니다.

### 동작 흐름

1. 기존 절차서, FAQ, 규칙을 재고 조사
2. 신규 멤버가 필요로 하는 정보를 우선순위순으로 정리
3. "먼저 읽어야 할 문서" 리스트를 작성
4. 내용이 오래되지 않았는지 정기 체크

### 활용 예시

- "새 엔지니어가 입사하면 이 5개 문서를 전달" 리스트
- 개발 환경 셋업 절차를 최신 상태로 유지
- 조직의 규칙 및 관행을 정리한 "하우스 룰" 정비

---

## 패턴 6: 문서 신선도 관리

### 하는 일

기존 문서의 내용이 현상과 괴리되지 않았는지 정기적으로 체크합니다.

### 동작 흐름

1. 전체 문서의 최종 업데이트일을 관리
2. 업데이트로부터 일정 기간(예: 3개월) 경과한 것을 리스트업
3. 내용과 현상을 비교하여 업데이트가 필요한지 판정
4. 업데이트가 필요한 것은 담당자에게 알림

### 활용 예시

- "다음 3개의 절차서가 3개월 이상 업데이트되지 않았습니다"
- "절차서 A에 기재된 명령이 버전업으로 변경되었습니다"
- "FAQ 답변 B가 현재 UI와 일치하지 않습니다"

### 최소 구성 (Anima 1개)

- 1개가 지식 관리 전반을 담당
- 주로 사람의 지시에 따라 절차서를 작성/업데이트
- FAQ 자동 답변도 겸임

### 권장 구성

- **지식 관리 담당**: 문서 작성, 업데이트, 신선도 관리
- 다른 Anima(개발, 감시 등)가 발견한 지견을 지식 관리 담당에게 전달
- 지식 관리 담당이 체계적으로 정리 및 보존

### 효과적인 운용 요령

- 처음부터 완벽을 목표로 하지 마세요. 문제가 발생하면 기록하는 것의 반복으로 키워갑니다
- "검색으로 찾을 수 있는 것"이 가장 중요합니다. 제목과 카테고리를 정성스럽게 붙이세요
- 절차서는 "실행하여 동작하는 것"을 확인한 후 공개하세요
- 정기적인 재고 조사로 진부화된 정보를 업데이트/삭제하세요
