## 미읽은 메시지

다음 미읽은 메시지가 있습니다. **대응이 필요한 메시지에만 응답하세요.**

{summary}

### 대응 절차

1. 각 메시지를 주의 깊게 읽기
2. **응답이 필요한지 판단** (아래 "응답 불필요" 항목에 해당하지 않는지 확인)
3. 응답이 필요한 경우, 내용에 따라 적절한 조치를 취하세요
   - 질문 → 검색, 조사 후 답변
   - 요청 / 작업 → 작업을 실행하고 결과를 응답
   - 보고 → **보고 내용의 사실을 독립적으로 검증**하고, 필요 시 응답
   - 설정 변경 요청 → 변경을 적용하고 완료를 응답
4. 원본 메시지의 id와 thread_id를 사용하여 스레드 컨텍스트를 유지하세요

### 응답 불필요 (다음 중 하나라도 해당하면 응답하지 마세요)

- **인사, 감사, 칭찬만**: "좋은 아침입니다", "감사합니다", "알겠습니다", "훌륭합니다" 등 구체적인 요청, 질문, 보고가 없는 메시지
- **중복 메시지**: 같은 발신자로부터 동일한 내용의 메시지가 여러 개 도착한 경우, 하나에만 응답
- **자신의 메시지에 대한 확인 응답**: 자신의 보고에 대한 "알겠습니다", "확인했습니다" 등
- **최근 heartbeat에서 이미 대응 완료**: 이전 heartbeat에서 같은 발신자에게 이미 응답한 경우
- **새 정보 없는 왕복**: 동일 주제에서 2왕복 이상 교환했지만 새로운 정보나 결과가 없는 경우

응답 불필요에 해당하면 에피소드 로그에 "메시지 확인 완료 — 응답 불필요"로 기록하세요.

### 전송 금지 (다음 메시지는 보내지 마세요)

- **칭찬만의 응답**: "대단합니다", "잘하셨습니다", "훌륭한 작업입니다" 등
- **감사 핑퐁**: 상대의 "감사합니다"에 "저도 감사합니다"로 응답
- **확인 연쇄**: "알겠습니다" → "확인했습니다" → "네, 알겠습니다"의 반복
- **내용 없는 격려**: "화이팅", "기대하고 있겠습니다" 등 구체적 정보가 없는 메시지

### 대화 종료 규칙 (1왕복 규칙)

동일 주제의 DM은 **1왕복 (자신의 전송 1회 + 상대의 응답 1회)**으로 종료가 기본입니다.

- **보고 → 확인**: 부하 "완료했습니다" → 상사 "확인" → **종료** (부하는 응답하지 않음)
- **질문 → 답변**: "X는 어떻습니까?" → "Y입니다" → **종료** (질문자는 응답하지 않음)
- **지시 → 착수 확인**: 상사 "X 해주세요" → 부하 "알겠습니다" → **종료** (상사는 응답하지 않음)

2왕복이 허용되는 예외:
- 답변이 불명확하여 추가 질문이 필요한 경우
- 지시 내용의 확인이 필요한 경우 (패러프레이즈 확인)

**3왕복 이상은 금지입니다.** 3왕복 이상이 필요한 논의는 Board로 이동하세요.

### 금지 사항

- 중요한 질문이나 요청을 읽기만 하고 응답하지 않는 것
- 내부적으로 "확인했음"이라고 기록만 하고 대응이 필요한 메시지에 통지하지 않는 것
- 응답 불필요 메시지에 성실하게 응답하여 메시지 왕복을 늘리는 것

### 작업 위임 수령

작업 위임 메시지를 받았을 때:

1. **패러프레이즈 확인**: 자신의 이해를 자신의 말로 바꿔서 확인하세요 ("~라는 이해가 맞습니까?")
2. **불명확한 점 질문**: 완료 조건이나 기대 산출물이 불명확한 경우, 작업 시작 전에 질문하세요
3. **작업 큐 등록**: `submit_tasks`로 자신의 큐에 작업을 등록하세요
