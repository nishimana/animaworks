1. **새로운 교훈 또는 패턴**: 에피소드에서 배운 것으로, 향후 판단에 도움이 되는 것
2. **기존 knowledge 업데이트**: 기존 knowledge 파일에 추가하거나 수정해야 할 내용
3. **새 knowledge 파일**: 새로운 주제로서 독립된 knowledge 파일이 필요한 경우

출력 형식:
## 기존 파일 업데이트
- Filename: knowledge/xxx.md
  추가 내용: (구체적인 내용을 Markdown 형식으로 작성)

## 새 파일 생성
- Filename: knowledge/yyy.md
  내용: (파일 전체 내용을 Markdown 형식으로 작성)

참고 사항:
- 다음 정보는 반드시 추출하세요:
  - 구체적인 설정 값, API 키, 인증 정보 저장 위치
  - 사용자 및 시스템 식별 정보 (ID, 이름, 역할)
  - 절차, 워크플로우, 프로세스 기록
  - 팀 구성, 역할 분담, 지휘 체계
  - 기술적 결정과 그 근거
- 완전히 동일한 내용의 반복만 스킵하세요
- 인사만 포함된 대화나 실질적 정보가 없는 교환은 knowledge화하지 마세요
- 기존 파일이 없는 경우, 모두 새 파일로 제안하세요
- 파일명은 주제를 명확히 나타내는 이름으로 하세요 (영숫자와 하이픈 권장)
